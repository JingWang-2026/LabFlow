# LabFlow

LabFlow 是一个局域网实验室批次协同工具，用于管理化学合成与生物测试进度。

## 启动

在服务器电脑上双击：

```text
启动LabFlow.bat
```

## 开机自启动

当前电脑已配置 Windows 登录后自动后台启动 LabFlow。自启动文件位于：

```text
C:\Users\31588\AppData\Roaming\Microsoft\Windows\Start Menu\Programs\Startup\LabFlow_AutoStart.vbs
```

它会调用：

```text
start_labflow_background.py
```

该脚本会先检查 `127.0.0.1:8080` 是否已经有服务在运行，避免重复启动。

如需取消自启动，删除启动文件夹里的 `LabFlow_AutoStart.vbs` 即可。

启动后，服务器电脑本机可访问：

```text
http://127.0.0.1:8080
```

同一办公室局域网内其他电脑访问：

```text
http://服务器电脑IP:8080
```

当前服务器电脑局域网地址是：

```text
http://172.16.1.71:8080
```

如果其他电脑打不开，请在服务器电脑上右键点击：

```text
开放LabFlow局域网访问.bat
```

选择“以管理员身份运行”。该脚本会放行 Windows 防火墙的 TCP 8080 端口。

查看服务器电脑 IP：

```powershell
ipconfig
```

通常找 `IPv4 地址`，形如 `192.168.x.x`。

## 默认账号

首次启动会自动创建这些账号。登录后请尽快修改密码。

| 角色 | 账号 | 默认密码 |
| --- | --- | --- |
| 总负责人 | leader | labflow123 |
| 化学部门 | chem1 / chem2 / chem3 | chem123 |
| 生物部门 | bio1 / bio2 / bio3 / bio4 / bio5 | bio123 |

总负责人可在页面右上角“账号”里重置其他账号密码。

## 权限

- 总负责人：创建/删除项目和批次，编辑全部字段，上传全部文件。
- 化学部门：编辑合成提交时间、合成完成时间，上传化合物信息文件。
- 生物部门：编辑生物测试开始时间、生物测试完成时间，上传生物原始数据文件和数据整理文档。
- 所有账号都可以查看全部项目、批次、状态和文件历史版本。

## 数据位置

- 数据库：`data/labflow.db`
- 上传文件：`uploads/`

建议定期备份整个 LabFlow 文件夹，至少备份 `data` 和 `uploads`。

## 删除与恢复

项目和批次删除后会进入回收站，不会立即物理删除。总负责人可在页面右上角“回收站”中恢复。

恢复规则：

- 如果项目被删除，需要先恢复项目。
- 项目恢复后，才能恢复该项目下的批次。
- 批次名称必须全系统唯一，包括回收站中的批次。

## 批次唯一性

- 批次编号可以重复。
- 批次名称不能为空，且必须全系统唯一。

## Windows 防火墙

如果其他电脑打不开页面，需要在服务器电脑 Windows 防火墙中允许 Python 在专用网络通信，或开放 TCP 端口 `8080`。

## 状态规则

- 没有合成提交时间：待合成
- 有合成提交时间，没有合成完成时间：合成中
- 有合成完成时间，没有生物测试开始时间：待测试
- 有生物测试开始时间，没有生物测试完成时间：测试中
- 有生物测试完成时间：已完成
